Overview

SOIL is a privacy-first, browser-based framework for on-device plant disease diagnosis on mobile phones. It combines lightweight crop-specific CNNs optimized via the Chroma-Sense architecture with TensorFlow.js to run inference entirely within the mobile browser — no app installation, no server upload, no privacy risk.

🌿
Core guarantee: All inference occurs within the browser. Image data never leaves the user's device. SOIL is the first framework to simultaneously achieve both cross-device compatibility and data privacy for plant disease diagnosis.
Smart Agriculture Data Privacy TensorFlow.js On-Device AI Cross-Platform

The Problem

Current plant disease diagnosis systems force farmers to choose between privacy and accessibility. Existing approaches fall into two categories — both with critical limitations.

Cloud-Based Apps
  • Images uploaded to remote servers
  • Serious data privacy & security risks
  • Eavesdropping & data tampering exposure
  • Requires reliable internet connectivity
  • Cross-platform compatible — but at a cost
Edge Device Apps
  • Requires dedicated hardware (Raspberry Pi, Jetson Nano)
  • Platform-specific: iOS & Android apps need separate builds
  • Users reluctant to install rarely-used apps
  • Privacy-preserving but not cross-platform
  • Impractical for everyday farmers
SOIL — The Solution
  • Runs entirely in the mobile browser — no installation required
  • Images never leave the device (on-device inference via TF.js)
  • Works on any modern smartphone: iOS, Android, desktop
  • Dynamic model loading: only the selected crop model is fetched
  • Browser-level caching reduces bandwidth after first use
3

SOIL Framework

SOIL has three tightly integrated components that together enable privacy-preserving, cross-platform plant disease diagnosis.

Chroma-Sense CNN

Lightweight CNN architecture that processes each color channel (R, G, B) independently through a shared feature extractor, then fuses the outputs via point-wise convolution. Designed specifically for browser-constrained memory and compute.

TensorFlow.js Runtime

Converts trained TensorFlow models into browser-executable format (.json + .bin shards). Automatically selects the optimal execution backend — WebGL (GPU), WASM, or pure JavaScript — ensuring functionality across all device tiers.

Dynamic Model Serving

Users select their crop type in the web UI. The corresponding crop-specific model is then fetched from the server and cached locally. Subsequent inferences use cached weights — minimizing bandwidth especially critical in rural areas.

Chroma-Sense Architecture

The model processes each RGB color channel independently through a shared convolutional encoder — a technique called serial multi-channel processing. This is motivated by the observation that plant disease patterns (rings, spots, rust, lesions) manifest distinctly within individual color channels.

Shared Feature Extractor

Each channel (256×256×1) passes through the same encoder:

channel_encoder.py
# Input: 256×256×1 (single channel) Conv2D(8, kernel=3, stride=2) # 128×128×8 SepConv2D(24, kernel=3, stride=2) # 64×64×24 SepConv2D(24, kernel=3, stride=2) # 32×32×24 SepConv2D(24, kernel=3, stride=2) # 16×16×24 SepConv2D(24, kernel=3, stride=2) # 8×8×24 SepConv2D(24, kernel=3, stride=2) # 4×4×24 SepConv2D(16, kernel=3, stride=2) # 2×2×16

Full Model Assembly

Three channel outputs are concatenated then fused:

full_model.py
# Input: 256×256×3 (RGB) R, G, B = split_channels(input) # Same encoder — shared weights feat_R = encoder(R) # 2×2×16 feat_G = encoder(G) # 2×2×16 feat_B = encoder(B) # 2×2×16 # Concatenate channel features fused = Concatenate([feat_R, feat_G, feat_B]) # 2×2×48 # Point-wise fusion x = Conv2D(16, kernel=1)(fused) # 2×2×16 x = GlobalMaxPool2D()(x) # 16 x = Dense(16, 'relu')(x) out = Dense(N, 'softmax')(x) # N classes
💡
Why SeparableConv2D? Depth-wise separable convolutions factor a standard convolution into depth-wise + point-wise steps, reducing parameters by ~8-9× with minimal accuracy loss. This is critical for fitting models in browser memory and achieving fast WebGL inference on mobile GPUs.

Disease-specific semantics (rings, dots, patches) were carefully preserved across train/validation splits to ensure models learn robust discriminating features rather than dataset artifacts. All images reshaped to 256×256 to match Chroma-Sense model input.

Grad-CAM Interpretability

Grad-CAM visualizations confirm that models attend to disease-relevant features — lesions, necrotic spots, rust patches, and discoloration — rather than background artifacts. This validates that Chroma-Sense learns meaningful disease semantics across all four crop types.

🔍
Apple & Tomato: Grad-CAM heat maps highlight edge rings, mosaic patterns, and septoria spots on the leaf surface.
🔍
Grape & Corn: Models focus on Esca lesion clusters, rust pustules, and blight spreading zones — consistent with expert agronomic diagnosis.
